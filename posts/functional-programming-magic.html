Functional programming is like magic

Like every magic, it's difficult and not for everyone

By not for everyone I mean, that if you struggle to understand functional programming you are not less intelligent. It just might mean that you think and solve problems in different way.

By it's difficult I mean exactly this. If you have several years of experience in object oriented programming, functional way will force you to change perspective and shift paradigms.

But what is functional programming?

This is not an easy question. Let me try to answer it.

Pure functions

Yes, this sounds reasonable, functional programming must have something to do with functions, right?

  • Pure function takes some value and returns some value only
  • Pure function depends only on it's arguments
  • Pure function is side effects free
public static Integer inc(Integer v) {
	return v+1;
}

This is nice function, beautiful in it's purity and simplicity.
